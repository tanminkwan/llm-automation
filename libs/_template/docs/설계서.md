## 1. 개요

요구사항의 FR/NFR 을 SOLID 원칙에 따라 다음과 같이 구현한다.

- **SRP** — `Greeter` 는 인사말 포맷팅만, `TemplateSettings` 는 환경변수 로딩만.
- **DIP** — `Greeter` 는 시계 의존성을 `Clock` Protocol 로 추상화하여 주입받는다.
- **LSP** — `Clock` Protocol 을 만족하는 어떤 구현도 대체 가능.

## 3. 인터페이스

```python
# src/_template/greeter.py
class Clock(Protocol):
    def now_iso(self) -> str: ...

class Greeter:
    def __init__(self, clock: Clock, prefix: str) -> None: ...
    def greet(self, name: str) -> str: ...      # ValueError if name == ""

# src/_template/settings.py
class TemplateSettings(BaseSettings):
    greeting_prefix: str  # default "Hello", env: TEMPLATE_GREETING_PREFIX
```

## 4. 핵심 시퀀스

```
caller
  │
  │ 1) settings = TemplateSettings()                  ← env 자동 로딩
  │ 2) greeter  = Greeter(clock=SystemClock(),
  │                       prefix=settings.greeting_prefix)
  │ 3) text     = greeter.greet("Alice")
  ▼
"[2026-04-15T16:00:00] Hello, Alice!"
```

본 모듈에는 `SystemClock` 구현을 포함하지 않는다 (운영 코드 아님). 호출측이
필요에 따라 자체 `Clock` 구현을 주입.

## 5. 데이터 모델

- `Greeter` 출력: `f"[{iso}] {prefix}, {name}!"` (str)
- `TemplateSettings.greeting_prefix`: str

## 9. 운영 / 배포 고려

- 본 모듈은 *배포* 대상 아님. `runtime` stage 빌드는 표준이 동작함을 증명하기 위함.
- 컨테이너 진입점은 단순 `python -c "from _template import Greeter; print('_template ok')"`.

## 10. SOLID 검토

| 원칙 | 적용 |
|---|---|
| SRP | `Greeter` 는 포맷팅만, `TemplateSettings` 는 설정만 |
| OCP | `Clock` Protocol 만 만족하면 새 구현 추가에 `Greeter` 수정 불필요 |
| LSP | `Clock` Protocol 만족 구현은 모두 대체 가능 (테스트로 검증) |
| ISP | `Clock` 은 `now_iso()` 한 메서드만 — 호출자가 필요한 만큼만 |
| DIP | `Greeter` 는 구체 시계가 아닌 `Clock` 추상화에 의존 |
