## 1. 개요

워크스페이스 표준(uv workspace · ruff · mypy · pytest · 4-stage Dockerfile · `Makefile` 표준 타깃 · pydantic-settings 기반 환경변수)이 **실제로 동작함을 증명** 하기 위한 최소 모듈. 외부 의존성 없이 SOLID·TDD·커버리지·multi-stage 이미지 빌드를 시연한다.

## 2. 배경 / 목적

- 표준이 문서로만 존재하면 작업자마다 해석 차이 발생.
- 돌아가는 **minimal example** 이 한 개 있어야 후속 모듈이 `cp -r` 로 안전하게 시작 가능.
- CI 도입 후에는 본 모듈이 **회귀 테스트** 역할 (표준 자체가 깨졌는지 감지).

## 3. 기능 요구사항 (FR)

| ID | 기능 | 설명 |
|---|---|---|
| FR-01 | `Greeter` 클래스 | DIP 시연 — `Clock` Protocol 주입, `greet(name)` 호출 시 `[<iso>] <prefix>, <name>!` 반환 |
| FR-02 | 빈 이름 거부 | `greet("")` 호출 시 `ValueError` |
| FR-03 | `TemplateSettings` | pydantic-settings 기반, `TEMPLATE_GREETING_PREFIX` env 로딩, 기본값 `Hello` |
| FR-04 | 환경변수 검증 | 잘못된 타입 입력 시 `ValidationError` |
| FR-05 | 표준 4-stage Dockerfile | base / builder / tester / runtime, runtime 이미지 ≤ 200MB |
| FR-06 | 표준 `Makefile` 7개 타깃 | `test` / `coverage` / `lint` / `format` / `build` / `image` / `clean` |

## 4. 비기능 요구사항 (NFR)

| ID | 항목 | 기준 |
|---|---|---|
| NFR-01 | 라인 커버리지 | ≥ 95% (목표 100%) |
| NFR-02 | 브랜치 커버리지 | ≥ 95% |
| NFR-03 | mypy strict | 통과 |
| NFR-04 | ruff check + format-check | 통과 |
| NFR-05 | runtime 이미지 크기 | ≤ 200MB |
| NFR-06 | 외부 의존성 | 없음 (in-process 만) |

## 5. 제약 / 가정

- C-01: `pydantic`, `pydantic-settings` 외 런타임 의존성 금지 (의도적 최소화).
- C-02: 비즈니스 가치 0. 본 모듈 자체는 운영에서 사용되지 않음.
