## 1. 개요

AI Agent CLI(OpenCode, Claude Code, Aider 등)를 **단일 인터페이스** (`AgentRunner` Protocol)로 추상화하는 라이브러리.
호출측(Celery Worker)은 구체 Agent CLI 를 모르고, 설정값(`AGENT_RUNNER_KIND`)만으로 backend 가 교체된다.

## 2. 배경 / 목적

- `architecture_test.md §1.2 T6` — OpenCode 외 도구(Claude Code, Aider, Cursor CLI 등)로 교체 가능하도록 `AgentRunner` 인터페이스로 분리.
- `architecture_test.md §6` — Tool(MCP) 등록 방식, 결과 diff 파싱, 종료 코드만 어댑터별로 다름 → 인터페이스 한 곳에서 흡수.
- Celery task 가 OpenCode subprocess 를 직접 호출하면 Agent CLI 교체 시 N 곳 동시 수정 → SOLID DIP 위반.
- 라이브러리화하여 wheel 로 배포 → 각 서비스 이미지 빌드 시 wheel 한 개만 끼우면 됨.

## 3. 기능 요구사항 (FR)

| ID | 산출물 / 기능 | 설명 |
|---|---|---|
| FR-01 | `AgentRunner` Protocol | `run(*, work_dir, prompt, files, tools, env) -> AgentResult` 시그니처 정의 (DIP 기반, `architecture_test.md §6`) |
| FR-02 | `ToolSpec` 데이터 모델 | MCP 도구 사양: `name`, `description`, `input_schema` (JSON Schema dict) |
| FR-03 | `AgentResult` 데이터 모델 | 실행 결과: `exit_code`, `stdout`, `stderr`, `changed_files`, `duration_seconds` |
| FR-04 | `OpenCodeRunner` 구현체 | `opencode run --prompt ... --tool ...` subprocess 실행. CLI 인자 조립 + 결과 파싱 |
| FR-05 | `ClaudeCodeRunner` 구현체 | `claude --print ...` subprocess 실행 (후보 — 인터페이스만 동일, 교체 가능성 검증) |
| FR-06 | `build_runner` 팩토리 | `build_runner("opencode") -> OpenCodeRunner`, `build_runner("claude-code") -> ClaudeCodeRunner`. 미등록 kind → `UnknownRunnerError` |
| FR-07 | `RunnerSettings` (pydantic-settings) | `AGENT_RUNNER_KIND`, `AGENT_TIMEOUT_SECONDS`, `AGENT_WORK_DIR_PREFIX` 환경변수 로딩 |
| FR-08 | 에러 모델 | `RunnerError` (base), `UnknownRunnerError` (미등록 kind), `AgentTimeoutError` (타임아웃 초과), `AgentExecutionError` (비정상 종료) |
| FR-09 | ToolSpec → CLI 인자 변환 | 어댑터별 tool 등록 형식 차이 흡수. OpenCode: `--tool <json>`, Claude Code: `--mcp-config <path>` |
| FR-10 | 타임아웃 제어 | subprocess 실행 시 `AGENT_TIMEOUT_SECONDS` 로 wall-clock 제한. 초과 시 `AgentTimeoutError` |

## 4. 비기능 요구사항 (NFR)

| ID | 항목 | 기준 |
|---|---|---|
| NFR-01 | 라인 커버리지 | ≥ 95% (목표 100%) |
| NFR-02 | 브랜치 커버리지 | ≥ 95% |
| NFR-03 | mypy strict | 통과 (`libs/` 전체 strict) |
| NFR-04 | ruff check + format-check | 통과 |
| NFR-05 | runtime 이미지 크기 | ≤ 200MB (외부 SDK 의존 없음) |
| NFR-06 | 외부 호출 격리 | 단위 테스트는 subprocess fake 만 사용. 실제 CLI 실행 0회 |
| NFR-07 | 하드코딩 금지 | CLI 경로/타임아웃/인자 등 의미 있는 상수 0개 (그라운드 룰 §7) |
| NFR-08 | 결정성 | fake subprocess 주입 시 호출 인자 → 결과가 결정적 |

## 5. 제약 / 가정

- C-01: 본 turn 에서 `OpenCodeRunner`, `ClaudeCodeRunner` 2종 구현. `AiderRunner` 는 OCP 로 가능성만 보장.
- C-02: 동기 인터페이스만 노출 (호출측 Celery worker 가 동기 친화적).
- C-03: Agent CLI 는 호스트에 설치 — 라이브러리 자체가 CLI 바이너리를 포함하지 않음.
- C-04: subprocess 실행이므로 stdout/stderr 캡처 기반. 구조화된 JSON 출력이 있으면 파싱, 없으면 raw text.
- A-01: OpenCode CLI 가 `opencode run` 서브커맨드와 `--prompt`, `--tool` 플래그를 지원한다고 가정.
- A-02: Claude Code CLI 가 `claude --print` 플래그와 `--mcp-config` 플래그를 지원한다고 가정.

## 6. 영향 범위

- **상류:** P00 워크스페이스 표준 (도구·디렉터리·Dockerfile 패턴).
- **하류:**
  - `services/celery-comment` (P05) — `AgentRunner.run()` 으로 코드 주석 Agent 실행
  - `services/celery-configaudit` (P05) — `AgentRunner.run()` 으로 Config 분석 Agent 실행
- 본 모듈의 **public API 변경** 시 위 2개 서비스 어댑터 갱신 필요. 인터페이스는 Phase 1 종료 시 고정.
