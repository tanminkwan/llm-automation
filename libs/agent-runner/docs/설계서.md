## 1. 개요

요구사항 §3 의 FR 을 SOLID 원칙에 따라 다음과 같이 구현한다.

- **SRP** — Protocol(인터페이스) / Runner(subprocess 실행) / Models(데이터) / Settings(env 로딩) / Errors(예외) / Factory(생성) 분리.
- **OCP** — 새 Agent CLI(Aider 등) 추가는 `AgentRunner` Protocol 만족 + registry 등록만으로 가능. 기존 코드 무수정.
- **LSP** — Protocol 만족 구현은 모두 `build_runner` 팩토리에서 대체 가능.
- **ISP** — `AgentRunner` 는 `run()` 단일 메서드만 노출. 내부 CLI 인자 조립/파싱 로직은 호출측에 노출되지 않음.
- **DIP** — 호출측(Celery)은 구체 Runner 가 아니라 Protocol 에 의존. 테스트는 fake 주입.

## 3. 인터페이스

### 3.1 Protocol (`protocols.py`)

```python
from pathlib import Path
from typing import Protocol
from .models import AgentResult, ToolSpec

class AgentRunner(Protocol):
    def run(
        self,
        *,
        work_dir: Path,
        prompt: str,
        files: list[Path],
        tools: list[ToolSpec],
        env: dict[str, str],
    ) -> AgentResult: ...
```

`architecture_test.md §6` 의 시그니처를 그대로 채용. keyword-only 인자로 안전성 확보.

### 3.2 데이터 모델 (`models.py`)

```python
from pathlib import Path
from pydantic import BaseModel, Field

class ToolSpec(BaseModel):
    """MCP 도구 사양 — Agent CLI 에 등록할 tool 정의."""
    name: str                                    # e.g. "search_codebase"
    description: str                             # 한 줄 설명
    input_schema: dict[str, object]              # JSON Schema dict
    server_url: str                              # MCP 서버 endpoint

class AgentResult(BaseModel):
    """Agent CLI 실행 결과."""
    exit_code: int                               # 프로세스 종료 코드
    stdout: str                                  # 표준 출력 (raw)
    stderr: str                                  # 표준 에러 (raw)
    changed_files: list[Path] = Field(default_factory=list)  # 변경된 파일 목록
    duration_seconds: float                      # 실행 소요 시간 (초)
```

> Q-01 결정: `AgentResult` 는 **raw stdout/stderr + 변경 파일 목록** 을 제공. structured diff 파싱은 호출측 책임 — Runner 는 CLI 실행에만 집중 (SRP).

### 3.3 Errors (`errors.py`)

```python
class RunnerError(Exception):
    """agent-runner 모든 예외의 base."""

class UnknownRunnerError(RunnerError):
    """build_runner 에 등록되지 않은 kind."""

class AgentTimeoutError(RunnerError):
    """subprocess 실행이 AGENT_TIMEOUT_SECONDS 초과."""

class AgentExecutionError(RunnerError):
    """Agent CLI 가 비정상 종료 (exit_code != 0)."""
    def __init__(self, message: str, result: AgentResult) -> None:
        super().__init__(message)
        self.result = result
```

### 3.4 Settings (`settings.py`)

```python
from pydantic_settings import BaseSettings, SettingsConfigDict

class RunnerSettings(BaseSettings):
    model_config = SettingsConfigDict(env_file=".env", extra="ignore")

    agent_runner_kind: str = Field(
        alias="AGENT_RUNNER_KIND", default="opencode"
    )
    agent_timeout_seconds: float = Field(
        alias="AGENT_TIMEOUT_SECONDS", default=300.0
    )
    agent_work_dir_prefix: str = Field(
```

Claude Code CLI 의 인자 형식은 OpenCode 와 다르지만, `AgentRunner` Protocol 을 만족하므로 호출측에는 차이 없음 (LSP).

### 3.8 Factory (`factory.py`)

```python
_REGISTRY: dict[str, type] = {
    "opencode": OpenCodeRunner,
    "claude-code": ClaudeCodeRunner,
}

def build_runner(kind: str, *, settings: RunnerSettings | None = None) -> AgentRunner:
    """kind 로 Runner 구현체 생성. 미등록 kind → UnknownRunnerError."""
    if kind not in _REGISTRY:
        raise UnknownRunnerError(f"Unknown runner kind: {kind!r}")
    settings = settings or RunnerSettings()
    return _REGISTRY[kind](settings=settings)

def register_runner(kind: str, cls: type) -> None:
    """OCP — 외부 Runner 등록."""
    _REGISTRY[kind] = cls
```

## 4. 핵심 시퀀스

```
caller (Celery Worker)
  │
  │ 1) settings = RunnerSettings()                    ← env 로딩
  │ 2) runner   = build_runner(settings.agent_runner_kind,
  │                            settings=settings)      ← factory
  │ 3) result   = runner.run(
  │                  work_dir=Path("/workspace/repo"),
  │                  prompt="Add docstrings...",
  │                  files=[Path("src/main.py")],
  │                  tools=[ToolSpec(name="search_codebase", ...)],
  │                  env={"CHAT_LLM_API_KEY": "sk-..."})
  │
  ├─► runner._build_command:
  │   - prompt, files, tools → CLI 인자 리스트 조립
  │   - ToolSpec → JSON 인라인 변환
  │
  ├─► _subprocess.run_cli:
  │   - subprocess.run(cmd, cwd, env, timeout)
  │   - stdout/stderr 캡처
  │   - 타임아웃 초과 → AgentTimeoutError
  │   - exit_code + stdout + stderr → AgentResult
  │
  ▼
AgentResult(exit_code=0, stdout="...", stderr="",
            changed_files=[...], duration_seconds=42.3)
```

## 9. 운영 / 배포 고려

- **배포 단위:** wheel (`uv build --wheel`). 후속 서비스 이미지는 builder stage 에서 wheel 설치.
- **컨테이너 이미지:** 표준 4-stage. runtime 은 `python -c "from agent_runner import build_runner; print('agent-runner ok')"` 스모크.
- **헬스체크:** 라이브러리이므로 별도 헬스체크 없음. 호출측 서비스가 책임.
- **자원:** 메모리 < 50MB (pydantic 만 로드, 외부 SDK 없음).

## 10. SOLID 검토

| 원칙 | 적용 |
|---|---|
| SRP | Protocol(인터페이스) / Runner(실행) / Models(데이터) / Settings(env) / Errors / Factory 모듈 분리. `_subprocess.py` 는 실행만, 인자 조립은 Runner 담당 |
| OCP | 새 Agent CLI 추가 = `AgentRunner` Protocol 만족 + `register_runner` 호출. `factory.py` / 호출측 코드 무수정 |
| LSP | `AgentRunner` 만족 구현은 모두 `build_runner` 에서 대체 가능 (테스트 T-24~T-27 로 검증) |
| ISP | `AgentRunner` 는 `run()` 단일 메서드. CLI 인자 조립(`_build_command`) 은 private — 호출측 불필요 |
| DIP | 호출측은 `AgentRunner` Protocol + `build_runner` 팩토리에만 의존. 구체 Runner import 불필요 |
