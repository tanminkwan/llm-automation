## 5. 95% 미달 시 사유

전체 98.47% 로 미달 아님. 개별 파일 중 2건의 브랜치 부분분기:
- `gateway.py:116` — backend factory 가 `None` 을 반환하는 이론적 경로의 else 가지. 실제 factory 계약상 None 반환 불가이나 방어 코드 유지.
- `openai_backend.py:140` — 재시도 루프가 generator 소진 없이 종료되는 이론적 경로의 terminal branch. 실제 tenacity 동작상 도달 불가.

두 경로 모두 방어적 성격이며, 의도적으로 unreachable-in-practice 로 남긴 가드다. 별도 테스트 추가 대신 주석/설계로 근거를 남겼다.

## 9. 회귀 / 기지 결함

검증 중 발견되어 해결된 이슈:

1. **tester 컨테이너에서 `ruff format --check` 실패** — 루트 `pyproject.toml` 의 `line-length = 100`/`select` 설정이 컨테이너 내부에선 안 보여 ruff 가 기본값(88 자)로 동작, 7개 파일에 대해 "would reformat" 을 리포트.
   - 해결: `libs/llm-gateway/pyproject.toml` 에 `[tool.ruff]`/`[tool.mypy]`/`[tool.pytest.ini_options]` 를 복제 고정. 컨테이너 단위 독립성(CLAUDE.md §1) 준수. 이후 module-local isort 가 `llm_gateway` 를 first-party 로 분류하여 테스트 파일 import 순서가 재정렬됨 — `make format` 으로 흡수.
2. **`conftest.py` S108 (insecure /tmp 사용)** — 최초 `monkeypatch.chdir("/tmp")` 사용 시 발생. 해결: pytest 의 `tmp_path` fixture 로 교체.
3. **gateway.chat() 이 backend 에 `model=None` 으로 전달** — backend 내부의 default_model 폴백은 있으나 테스트가 profile model 기대. 해결: `gateway.chat()` 에서 `model = model or profile.model` 을 선계산하여 의도 명시.
4. **`cov` 측정 경로 누락 방지** — editable 설치(`pip install -e .`) 로 src/ 경로 기준 측정. (P00 에서 해결된 패턴 재사용)

미해결 이슈: 없음.
