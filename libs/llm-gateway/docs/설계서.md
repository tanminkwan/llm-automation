## 1. 개요

요구사항 §3 의 FR 을 SOLID 원칙에 따라 다음과 같이 구현한다.

- **SRP** — Backend(외부 호출) / Gateway(alias 라우팅) / Settings(env 로딩) / Errors(예외 모델) / Retry(재시도 정책) 분리.
- **OCP** — 새 backend(Anthropic, Azure ...) 추가는 `ChatBackend` Protocol 만족 + factory 등록만으로 가능.
- **LSP** — Protocol 만족 구현은 모두 `LLMGateway` 에 주입 가능.
- **ISP** — `ChatBackend` 와 `EmbeddingBackend` 는 분리. chat 만 필요한 호출자가 embedding API 를 알 필요 없음.
- **DIP** — `LLMGateway` 는 구체 backend 가 아니라 Protocol 에 의존. 테스트는 fake 주입.

## 3. 인터페이스

### 3.1 Protocol (`protocols.py`)

```python
from typing import Protocol
from .models import ChatMessage, ChatResponse, EmbeddingResponse

class ChatBackend(Protocol):
    def chat(
        self,
        messages: list[ChatMessage],
        *,
        model: str,
        temperature: float | None = None,
        max_tokens: int | None = None,
    ) -> ChatResponse: ...

class EmbeddingBackend(Protocol):
    def embed(self, texts: list[str], *, model: str) -> EmbeddingResponse: ...
```

### 3.2 데이터 모델 (`models.py`)

```python
from pydantic import BaseModel, Field

class ChatMessage(BaseModel):
    role: Literal["system", "user", "assistant", "tool"]
    content: str

class TokenUsage(BaseModel):
    prompt_tokens: int = 0
    completion_tokens: int = 0
    total_tokens: int = 0

class ChatResponse(BaseModel):
    content: str
    model: str
    finish_reason: str
    usage: TokenUsage = Field(default_factory=TokenUsage)

class EmbeddingResponse(BaseModel):
    vectors: list[list[float]]
    model: str
    dim: int                # 벡터 차원 (Qdrant collection 생성에 사용)
```

> Q-01 결정: raw provider response 는 노출하지 않음. 호출측이 필요로 하는 필드만 안정적인 스키마로 박제 — backend 변경이 호출측에 새지 않게.

### 3.3 Errors (`errors.py`)

```python
class GatewayError(Exception): ...
class UnknownProfileError(GatewayError): ...
class MissingCredentialsError(GatewayError): ...
class BackendCallError(GatewayError):
```

### 3.6 Retry (`retry.py`)

```python
def build_retry(timeout_seconds: float, max_retries: int) -> tenacity.Retrying:
    """RetryError 라이브러리를 통해 재시도 정책 빌드.

    - retry: APIConnectionError, RateLimitError, InternalServerError (5xx)
    - 비대상: BadRequestError, AuthenticationError (4xx 의도된 실패)
    - wait: exponential backoff (min=1s, max=10s)
    - stop: stop_after_attempt(max_retries)
    """
```

> Q-02 결정: openai SDK 가 던지는 예외 중 **transient** (네트워크/5xx/rate limit) 만 재시도. 4xx 인증/요청오류는 즉시 raise.

```python
        # alias 미등록 → UnknownProfileError
```

> factory 주입은 OCP/DIP — 테스트는 fake backend 를 주입, production 에서는 vLLM 어댑터를 주입할 수 있다 (단, OpenAI 호환이라 기본 구현 그대로 사용 권장).

## 4. 핵심 시퀀스

```
caller
  │
  │ 1) settings = GatewaySettings()                 ← env 로딩
  │ 2) gateway  = LLMGateway(settings)              ← 기본 OpenAI factory
  │ 3) resp     = gateway.chat("chat-llm",
  │                            messages=[...])
  │
  ├─► gateway: alias 검증 → CHAT_ALIASES 멤버?
  │   ├─ no  → UnknownProfileError
  │   └─ yes → settings.profile("chat-llm") → ProfileSettings
  │           ├─ api_key == "" → MissingCredentialsError
  │           └─ ok → factory(profile) → OpenAIChatBackend
  │
  ├─► OpenAIChatBackend.chat:
  │   - tenacity Retrying (timeout, max_retries)
  │   - openai.OpenAI(base_url, api_key).chat.completions.create(...)
  │   - 응답 → ChatResponse(content, model, finish_reason, usage)
  │   - 재시도 모두 실패 → BackendCallError
  ▼
ChatResponse
```

## 9. 운영 / 배포 고려

- **배포 단위:** wheel (`uv build --wheel`). 후속 서비스 이미지는 builder stage 에서 wheel 설치.
- **컨테이너 이미지:** 표준 4-stage. runtime 은 `python -c "from llm_gateway import LLMGateway; print('llm-gateway ok')"` 스모크.
- **헬스체크:** 라이브러리이므로 별도 헬스체크 없음. 호출측 서비스가 책임.
- **자원:** 메모리 < 100MB (openai SDK + pydantic 만 로드).

## 10. SOLID 검토

| 원칙 | 적용 |
|---|---|
| SRP | Backend(외부 호출) / Gateway(라우팅) / Settings(env) / Errors / Retry / Models 모듈 분리 |
| OCP | 새 backend 추가 = `ChatBackend`/`EmbeddingBackend` Protocol 만족 + factory 주입. `LLMGateway` 코드 무수정 |
| LSP | `ChatBackend` 만족 구현은 모두 `LLMGateway` 에서 대체 가능 (테스트로 검증) |
| ISP | `ChatBackend` 와 `EmbeddingBackend` 분리 — chat 만 쓰는 호출자가 embedding 메서드를 알 필요 없음 |
| DIP | `LLMGateway` 는 구체 클래스가 아닌 Protocol + factory 에 의존 |

## 11. 미해결 / 결정 종결

| ID (요구사항) | 결정 |
|---|---|
| Q-01 raw response 노출 | **비공개**. `ChatResponse`/`EmbeddingResponse` 로 박제 — backend 변경 격리 |
| Q-02 재시도 대상 예외 | **transient 만** (`APIConnectionError`, `RateLimitError`, `InternalServerError`). 4xx 즉시 raise |
| Q-03 EmbeddingResponse 구성 | **vectors + model + dim**. 원문 echo 없음 (호출측이 이미 가지고 있음) |
