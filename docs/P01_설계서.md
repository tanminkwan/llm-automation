## 1. 개요

Phase 1 은 두 개의 라이브러리(`libs/llm-gateway`, `libs/agent-runner`) 로 구성된다. 두 라이브러리 모두 P00 의 표준(uv workspace · ruff · mypy strict · pytest · 4-stage Dockerfile · pydantic-settings) 을 그대로 따른다. 본 문서는 **Phase 공통 결정** 만 다루고, 라이브러리별 상세 인터페이스는 각 모듈 `docs/설계서.md` 에서 정의한다.

## 2. Phase 1 디렉터리 추가 분

```
libs/
├── _template/                  ← P00 (불변)
├── llm-gateway/                ← FR-01 (본 턴)
│   ├── pyproject.toml
│   ├── Dockerfile              # 4-stage (P00 표준)
│   ├── Makefile                # 표준 타깃 (test/coverage/lint/format/build/image/clean)
│   ├── README.md
│   ├── docs/{요구사항,설계서,테스트결과서}.md
│   ├── src/llm_gateway/
│   └── tests/
└── agent-runner/               ← FR-02 (후속 턴)
    └── (동일 구조)
```

루트 `Makefile` `MEMBERS` 에 `libs/llm-gateway`, `libs/agent-runner` 가 phase 종료 시 모두 추가된다. 본 턴은 `libs/llm-gateway` 까지만 추가.

## 3. 공통 컨벤션

### 3.1 패키지명 ↔ 디렉터리명

| 디렉터리 | uv 패키지명 | import 경로 | Docker 이미지 |
|---|---|---|---|
| `libs/llm-gateway` | `llm-gateway` | `llm_gateway` | `llm-gateway` |
| `libs/agent-runner` | `agent-runner` | `agent_runner` | `agent-runner` |

> P00 의 `_template` 은 밑줄 시작이라 `seed-template` 으로 우회했지만, Phase 1 부터는 디렉터리·패키지·이미지명이 모두 같음.

### 3.2 환경변수 prefix

| 라이브러리 | env prefix | 비고 |
|---|---|---|
| `llm-gateway` | 없음 (직접 키) | `architecture_test.md §12.2` 의 표준 키 그대로 — `CHAT_LLM_*`, `REASONING_LLM_*`, `EMBEDDING_*` |
| `agent-runner` | `AGENT_` | `AGENT_KIND` (opencode/claude/aider), `AGENT_BIN_PATH` 등 |

llm-gateway 가 env prefix 를 두지 않는 이유: 표준 alias 가 이미 그룹핑(`CHAT_LLM_`, `REASONING_LLM_`, `EMBEDDING_`) 역할을 하므로 중복 prefix 불필요.

### 3.3 미해결 항목 결정 (P01_요구사항.md §9)

| ID | 결정 | 근거 |
|---|---|---|
| Q-01 OpenAI SDK | **`openai>=1.40` 동기 클라이언트** | OpenAI 호환 (`base_url` 변경으로 vLLM 도 흡수). httpx 직접 구현은 retry/SSE 처리를 재발명 → 비용. async 는 호출측이 `asyncio.to_thread` 로 흡수 가능 |
| Q-02 재시도/타임아웃 | **라이브러리 내장 (`tenacity` 또는 SDK 옵션)** + **호출측이 환경변수로 조정** | 호출측마다 재구현 방지. 기본값은 `architecture_test.md §1.3` 재현성 기준에 맞춰 보수적(timeout 60s, retries 3, exp backoff) |
| Q-03 설정 파일 | **환경변수 only (Phase 1 한정)** | `config/llm.yaml` 은 P02 `infra/` 합류 시 도입 검토. Phase 1 은 라이브러리 단위라 yaml 파서 의존성 추가는 과함. alias 자체는 코드 상수로 정의 (`chat-llm`/`reasoning-llm`/`embedding`) |
| Q-04~Q-05 | agent-runner 설계서에서 결정 (후속 턴) | — |

## 4. 의존성 (라이브러리 공통)

| 라이브러리 | 런타임 | 개발 |
|---|---|---|
| `llm-gateway` | `pydantic>=2.7`, `pydantic-settings>=2.4`, `openai>=1.40`, `tenacity>=9` | 워크스페이스 dev group (pytest, ruff, mypy) + `pytest-mock` |
| `agent-runner` | `pydantic>=2.7`, `pydantic-settings>=2.4` (subprocess 는 표준 라이브러리) | 동일 |

`openai` 는 `httpx` 를 transitive 의존으로 가져온다. tenacity 도입은 최소 (외부 호출 라이브러리에 한해 정당화).

## 5. 검증 시나리오 (Phase DoD 매핑)

| DoD 항목 | 검증 방법 |
|---|---|
| 모듈별 4-단계 산출물 존재 | `libs/<m>/docs/{요구사항,설계서,테스트결과서}.md` git ls-files |
| 표준 타깃 통과 | `make -C libs/<m> lint coverage image` 종료 코드 0 |
| 커버리지 ≥ 95% | `pytest --cov-fail-under=95` 종료 코드 |
| sample.env 동기화 | 본 문서 §3.2 ↔ 각 모듈 설계서 ↔ 루트 sample.env 수동 grep |
| 루트 Makefile MEMBERS | `grep "libs/llm-gateway" Makefile`, `grep "libs/agent-runner" Makefile` |

수치(테스트 케이스 수 / 커버리지 / 이미지 크기) 는 `docs/P01_테스트결과서.md` 에 모듈별 요약 + 각 모듈 자체 결과서 링크.

## 6. 운영 / 배포 고려

- 두 라이브러리 모두 **wheel 산출물**. 후속 서비스 이미지의 builder stage 에서 `pip install <wheel>` 로 흡수.
- 라이브러리 자체 runtime 이미지는 *표준 검증용 (스모크)*. 배포 대상 아님 — `_template` 과 동일 패턴.
