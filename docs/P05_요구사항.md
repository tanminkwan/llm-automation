## 1. 개요

Phase 5 는 테스트 환경 워크플로우의 진입점 3개 서비스를 구현한다.

- **flask-webhook**: Gitea push webhook 수신 → HMAC 검증 → 변경 파일 분류 → Celery task 발행
- **celery-comment**: `comment_queue` 소비 → 리포지토리 clone → RAG 기반 프롬프트 생성 → AgentRunner 실행 → 코드 주석 추가 → commit/push
- **celery-configaudit**: `configaudit_queue` 소비 → Agent loop (LLM + configaudit-mcp tool call) → 분석 보고서 작성

## 3. 기능 요구사항 요약

### 3.1 flask-webhook

| ID | 기능 | 설명 |
|---|---|---|
| FW-FR-01 | Gitea push webhook 수신 | `POST /webhook/push` 엔드포인트 |
| FW-FR-02 | HMAC-SHA256 검증 | `X-Gitea-Signature` 헤더 vs 공유 시크릿 |
| FW-FR-03 | 중복 방지 | `X-Gitea-Delivery` 기반 TTL 캐시 (멱등키) |
| FW-FR-04 | 변경 파일 분류 | Java → `comment`, http.m → `configaudit`, 혼합 → 양쪽 |
| FW-FR-05 | Celery task 발행 | `comment.process` → `comment_queue`, `configaudit.process` → `configaudit_queue` |
| FW-FR-06 | 헬스체크 | `GET /health` |

### 3.2 celery-comment

| ID | 기능 | 설명 |
|---|---|---|
| CC-FR-01 | comment_queue 소비 | Celery task `comment.process` |
| CC-FR-02 | 리포지토리 clone | Git clone → 작업 브랜치 |
| CC-FR-03 | RAG 기반 프롬프트 | RAG MCP 에서 유사 코드/주석 예시 검색 → 프롬프트 포함 |
| CC-FR-04 | AgentRunner 실행 | LLM agent 가 Java 코드에 주석 추가 |
| CC-FR-05 | commit/push | 결과를 git commit → push |
| CC-FR-06 | 결과 반환 | CommentResult (성공/실패, 변경 파일 수) |

### 3.3 celery-configaudit

| ID | 기능 | 설명 |
|---|---|---|
| CA-FR-01 | configaudit_queue 소비 | Celery task `configaudit.process` |
| CA-FR-02 | Agent loop | LLM ↔ tool_call 반복 (max_iterations 제한) |
| CA-FR-03 | MCP tool call | `get_config_context` 호출 → configaudit-mcp 연동 |
| CA-FR-04 | 분석 보고서 생성 | LLM 최종 응답 → AnalysisReport → Markdown 파일 |
| CA-FR-05 | 결과 반환 | report_path, summary 포함 dict |
