## 5. `_template` 모듈 설계 (Q-01 결정)

### 5.1 도메인 — `Greeter`

표준이 실제로 동작함을 증명하는 **최소 도메인**. 외부 의존성 없이 SOLID/TDD/커버리지를 검증할 수 있어야 함.

### 5.2 인터페이스 (SOLID 적용 예시)

```python
# src/_template/greeter.py
from typing import Protocol

class Clock(Protocol):                      # DIP — 시간 의존성 추상화
    def now_iso(self) -> str: ...

class Greeter:                              # SRP — 인사말 생성만
    def __init__(self, clock: Clock, prefix: str) -> None:
        self._clock = clock
        self._prefix = prefix

    def greet(self, name: str) -> str:
        if not name:
            raise ValueError("name must not be empty")
        return f"[{self._clock.now_iso()}] {self._prefix}, {name}!"
```

```python
# src/_template/settings.py
from pydantic import Field
from pydantic_settings import BaseSettings, SettingsConfigDict

class TemplateSettings(BaseSettings):       # 그라운드 룰 §7
    model_config = SettingsConfigDict(env_prefix="TEMPLATE_", env_file=".env")
    greeting_prefix: str = Field(default="Hello", description="인사말 접두어")
```

→ "직접 `os.environ` 읽지 않음", "테스트는 `monkeypatch.setenv` 로 주입" 의 reference.

### 5.3 설정 항목 표 (그라운드 룰 §7)

| 키 | 의미 | 기본값 | 필수 | 민감 |
|---|---|---|---|---|
| `TEMPLATE_GREETING_PREFIX` | 인사말 접두어 | `Hello` | ❌ | ❌ |

`sample.env` 에 동일 키 미러링.

## 6. 표준 multi-stage Dockerfile (`libs/_template/Dockerfile`)

```dockerfile
# syntax=docker/dockerfile:1.7
ARG PYTHON_VERSION=3.12

# ─── 1) base ────────────────────────────────────────────
FROM python:${PYTHON_VERSION}-slim AS base
ENV PYTHONDONTWRITEBYTECODE=1 PYTHONUNBUFFERED=1
RUN useradd -m -u 1000 app
WORKDIR /app

# ─── 2) builder ─────────────────────────────────────────
FROM python:${PYTHON_VERSION} AS builder
RUN pip install --no-cache-dir uv
WORKDIR /build
COPY pyproject.toml ./
COPY src ./src
RUN uv build --wheel --out-dir /build/dist

# ─── 3) tester (CI 전용) ────────────────────────────────
FROM builder AS tester
COPY tests ./tests
RUN uv pip install --system /build/dist/*.whl pytest pytest-cov ruff mypy
RUN ruff check src tests \
 && ruff format --check src tests \
 && mypy src \
 && pytest --cov=src --cov-branch --cov-fail-under=95

# ─── 4) runtime (배포) ──────────────────────────────────
FROM base AS runtime
COPY --from=builder /build/dist/*.whl /tmp/
RUN pip install --no-cache-dir /tmp/*.whl && rm -rf /tmp/*.whl
USER app
CMD ["python", "-c", "from _template.greeter import Greeter; print('ok')"]
```

빌드 명령:
- `docker build --target tester -t _template:test .` → 그라운드 룰 검증
- `docker build --target runtime -t _template:latest .` → 배포 이미지

## 7. 표준 `Makefile`

### 7.1 멤버용 (`libs/_template/Makefile`)

```makefile
.PHONY: test coverage lint format build image clean

UV ?= uv
SVC := _template

test:
	$(UV) run pytest

coverage:
	$(UV) run pytest --cov=src --cov-branch --cov-report=term-missing --cov-fail-under=95

lint:
	$(UV) run ruff check src tests
	$(UV) run ruff format --check src tests
	$(UV) run mypy src

format:
	$(UV) run ruff format src tests
	$(UV) run ruff check --fix src tests

build:
	$(UV) build --wheel

image:
	docker build --target runtime -t $(SVC):latest .

image-test:
	docker build --target tester -t $(SVC):test .

clean:
	rm -rf dist .pytest_cache .coverage htmlcov .ruff_cache .mypy_cache
```

### 7.2 루트 (`Makefile`) — Q-02 결정

**고정 list 채택** (동적 `find` 보다 명시적 + 새 모듈 추가가 자연스럽게 PR 게이트에 노출됨).

```makefile
MEMBERS := libs/_template
# 후속 phase 마다 멤버 추가:
# MEMBERS += libs/llm-gateway libs/agent-runner
# MEMBERS += services/ribalso-mock ...

.PHONY: test-all coverage-all lint-all image-all clean-all

test-all:      ; @for m in $(MEMBERS); do $(MAKE) -C $$m test || exit 1; done
coverage-all:  ; @for m in $(MEMBERS); do $(MAKE) -C $$m coverage || exit 1; done
lint-all:      ; @for m in $(MEMBERS); do $(MAKE) -C $$m lint || exit 1; done
image-all:     ; @for m in $(MEMBERS); do $(MAKE) -C $$m image || exit 1; done
clean-all:     ; @for m in $(MEMBERS); do $(MAKE) -C $$m clean; done
```

## 10. 테스트 전략 (TDD 케이스)

`_template` 모듈에 대해 다음 케이스를 **테스트 먼저** 작성:

| ID | 대상 | 케이스 |
|---|---|---|
| T-01 | `Greeter.greet` | 정상 입력 → `[<iso>] <prefix>, <name>!` |
| T-02 | `Greeter.greet` | 빈 이름 → `ValueError` |
| T-03 | `Greeter` | `Clock` 의존성 주입 검증 (Fake Clock) |
| T-04 | `TemplateSettings` | 기본값 (`Hello`) 적용 |
| T-05 | `TemplateSettings` | `monkeypatch.setenv("TEMPLATE_GREETING_PREFIX", "안녕")` → 반영 |
| T-06 | `TemplateSettings` | 잘못된 타입 환경값 → 검증 오류 |

기대 커버리지: 라인 100% / 브랜치 100% (모듈이 작아 95% 여유 큼).

## 11. 검증 시나리오 (P00 DoD 매핑)

| DoD 항목 | 검증 방법 |
|---|---|
| 산출물 존재 | `git status` + `tree` 로 디렉터리 확인 |
| 표준 타깃 통과 | `make -C libs/_template lint coverage image` |
| 커버리지 ≥ 95% | `pytest --cov-fail-under=95` 의 종료 코드 |
| 이미지 크기 ≤ 200MB | `docker images _template:latest --format '{{.Size}}'` |
| sample.env ↔ 설정표 동기화 | 수동 grep + 본 문서 §5.3 비교 |
| 테스트결과서 작성 | `docs/P00_테스트결과서.md` 존재 + 수치 기재 |

수치는 모두 `P00_테스트결과서.md` 에 기록.
