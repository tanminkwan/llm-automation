## 1. 개요

Phase 4 는 Config 변경 분석 워크플로우(Workflow B)의 핵심 MCP 서비스인 `configaudit-mcp` 를 구현한다. AI Agent 가 호출하는 `get_config_context` 도구를 통해 dev/stage/prod 3개 환경의 config 를 조회하고, 3-way diff + 이상 패턴 사전 탐지 결과를 구조화된 JSON 으로 반환한다.

## 3. 기능 요구사항 요약

| ID | 기능 | 설명 |
|---|---|---|
| FR-01 | `get_config_context` MCP tool | `work_id` + 선택적 `case` 입력 → 3-way diff 컨텍스트 반환 |
| FR-02 | 리발소 API 3환경 조회 | dev/stage/prod `GET /config/httpm` 호출 |
| FR-03 | 환경별 before→after diff | unified diff 생성 |
| FR-04 | 환경 간 after 비교 | dev↔stage, stage↔prod diff |
| FR-05 | 이상 패턴 탐지 | 1개 환경만 나머지와 다를 때 anomaly 보고 |
| FR-06 | 결과 스키마 안정화 | Pydantic 모델 기반 JSON 응답 |
| FR-07 | 헬스체크 | `GET /health` |
