### 4.1 Phase 6 실행 중 발견/수정 이슈

| # | 증상 | 근본원인 | 수정 |
|---|---|---|---|
| 1 | `flask-webhook` 컨테이너 빌드 실패 (`tool.uv.sources` workspace 해석 불가) | 런타임 컨테이너에서 workspace 미가용 | `pip install --no-deps . && uv build --no-sources --wheel` 패턴으로 전환 |
| 2 | `celery` / `uvicorn` 바이너리 not found | site-packages 만 복사, `/usr/local/bin/*` 누락 | CMD 를 `python -m celery`, `python -m uvicorn --factory` 로 교체 |
| 3 | python 기반 서비스 healthcheck 실패 | `python:3.12-slim` 에 `curl` 부재 | `python -c "import urllib.request; ..."` 로 교체 |
| 4 | `qdrant`, `gitea` healthcheck 실패 | 해당 이미지에 `curl` 부재 | bash `/dev/tcp` 포트 체크로 교체 |
| 5 | 스택 서비스가 "scheduling constraints not satisfied" | single-node 에서 `role=manager`, `role=worker` 라벨이 서로 덮어씀 | 별도 키 `node-role.manager=true`, `node-role.worker=true` 사용 |
| 6 | `celery-configaudit` 이 `/tools/get_config_context` 502 | `LEEBALSO_MOCK_URL` env 가 settings 에 없음 (`leebalso_base_url` 로 읽음) | stack.yml 에서 `LEEBALSO_BASE_URL=...` 로 수정 |
| 7 | 시나리오 B LLM 이 404 로 보고 실패 | task.py 가 `case = work_id` 로 전체를 케이스에 대입 | `work_id.rsplit(":", 1)[-1]` 로 케이스 추출 |
| 8 | 시나리오 B report JSON 스키마 불일치 | report_writer 가 Markdown 만 기록 | 동일 basename `.json` sidecar 추가 (schema-conformant) |
| 9 | 컨테이너가 쓴 결과 파일을 호스트 pytest 가 못 읽음 | 명명 volume 사용 | `FIXTURE_{SOURCE,RESULT}_DIR`, `REPORT_OUTPUT_DIR`, `COMMENT_WORK_DIR` 를 bind-mount (호스트=컨테이너 동일 경로) |
| 10 | `make e2e` 재실행 시 시나리오 A 가 `fixture source not found` 로 실패 | `stack_up.sh` 가 `scenario_A/before/` 의 내용만 `scenario_A/` 로 평탄화 복사 — `clone_url=fixture://scenario_A/before` 와 경로 불일치 | `cp -r .../scenario_A/before/. scenario_A/` 를 `cp -r .../scenario_A/before scenario_A/` 로 교체해 디렉터리 구조 보존 |
