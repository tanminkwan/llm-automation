### 2.4 `fixture_git.py` (Phase 6)

```python
class FixtureGitClient:
    """파일 복사 기반 GitClient — E2E / 로컬 테스트용.

    - clone(url='fixture://<key>', dest, ref):
        {source_root}/<key>/ 트리를 dest 로 복사 (dest 이미 존재 시 덮어쓰기).
        ref 는 기록용 (실제 checkout 없음).
    - add_commit_push(repo_dir, message, branch):
        repo_dir 전체 트리를 {result_root}/<branch>/<UTC timestamp>/ 로 스냅샷.
        snapshot 내부에 manifest.json (message, branch, timestamp, source_repo_dir) 기록.
    """
    FIXTURE_SCHEME = "fixture://"
    MANIFEST_NAME = "manifest.json"
```

### 2.5 `factory.py` (Phase 6)

```python
def build_git_client(settings: CommentSettings) -> GitClient:
    if settings.git_client == "fixture":
        return FixtureGitClient(settings.fixture_source_dir, settings.fixture_result_dir)
    return SubprocessGitClient(author=settings.git_commit_author)
```

운영 기본은 `subprocess`. 알 수 없는 값도 `subprocess` 로 fail-safe 처리.

## 3. 설정 항목

| 키 (env) | 의미 | 기본값 | 필수 | 민감 |
|---|---|---|---|---|
| `CELERY_BROKER_URL` | Redis broker | `redis://localhost:6379/0` | N | N |
| `CELERY_RESULT_BACKEND` | Redis result | `redis://localhost:6379/1` | N | N |
| `COMMENT_WORK_DIR` | 임시 작업 디렉터리 | `/tmp/celery-comment` | N | N |
| `RAG_MCP_URL` | RAG MCP 서비스 URL | `http://localhost:9001` | N | N |
| `AGENT_RUNNER_KIND` | AgentRunner 종류 | `opencode` | N | N |
| `AGENT_TIMEOUT_SECONDS` | Agent 실행 타임아웃 | `300` | N | N |
| `GIT_COMMIT_AUTHOR` | 커밋 author | `AI Bot <ai-bot@local>` | N | N |
| `GIT_CLIENT` | GitClient 구현 (`subprocess`/`fixture`) | `subprocess` | N | N |
| `FIXTURE_SOURCE_DIR` | fixture clone 소스 루트 | `/fixtures/source` | `GIT_CLIENT=fixture` 시 Y | N |
| `FIXTURE_RESULT_DIR` | fixture 스냅샷 결과 루트 | `/fixtures/results` | `GIT_CLIENT=fixture` 시 Y | N |
