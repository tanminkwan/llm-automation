## 1. 개요

Production 의 리발소 REST API 를 대체하는 **결정성 보장 Mock 서버**. 동일 요청에 대해 항상 동일한 config (before/after) 를 반환하며, 응답 데이터는 파일시스템 fixture 에서 로드한다.

## 2. 배경 / 목적

- `architecture_test.md §7` — 실제 리발소 서버 대신 테스트용 가상 서버(Mock) 를 컨테이너로 기동.
- `architecture_test.md §1.2 T7` — 동일 요청에 대해 항상 동일한 (변경 전/후) config 두 벌을 return.
- ConfigAudit MCP (P04) 가 이 mock 을 호출하여 3-way diff 를 생성 → 결정적 fixture 가 없으면 E2E 테스트가 비결정적.
- FastAPI 기반 단일 컨테이너. 자체 코드 + fixture 파일만 포함.

## 3. 기능 요구사항 (FR)

| ID | 산출물 / 기능 | 설명 |
|---|---|---|
| FR-01 | `GET /config/httpm` | 쿼리: `env=dev\|stage\|prod`, `case=<case-id>` (선택, 기본 `case-001`). 응답: JSON `{ env, before, after, meta }` |
| FR-02 | 결정성 보장 | 동일 `(env, case)` 요청 N회 → byte-equal JSON 응답 |
| FR-03 | 픽스처 기반 응답 | `fixtures/<case>/{env}.before.httpm`, `fixtures/<case>/{env}.after.httpm` 파일에서 로드 |
| FR-04 | 에러 처리 — env 누락 | `env` 쿼리 파라미터 미전달 → HTTP 422 |
| FR-05 | 에러 처리 — 미지 case | 존재하지 않는 case 디렉터리 → HTTP 404 |
| FR-06 | 에러 처리 — 미지 env | case 는 존재하나 해당 env fixture 없음 → HTTP 404 |
| FR-07 | `GET /health` | 헬스체크 엔드포인트. `{ "status": "ok" }` 반환 |
| FR-08 | 설정 주입 | `LEEBALSO_FIXTURES_DIR`, `LEEBALSO_HOST`, `LEEBALSO_PORT` 환경변수로 픽스처 경로/호스트/포트 조정 |

## 5. 제약 / 가정

- C-01: fixture 파일은 빌드 시 이미지에 포함. 런타임 동적 추가 미지원.
- C-02: 단일 케이스(`case-001`)부터 시작. 추가 케이스는 fixture 디렉터리 추가만으로 확장 (OCP).
- C-03: 인증/인가 없음 — 내부 테스트 환경 전용.
- A-01: fixture 파일은 UTF-8 텍스트.
