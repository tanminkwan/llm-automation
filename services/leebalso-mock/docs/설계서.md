## 1. 개요

요구사항 §3 의 FR 을 SOLID 원칙에 따라 다음과 같이 구현한다.

- **SRP** — Router(라우팅) / FixtureLoader(파일 I/O) / Models(응답 직렬화) / Settings(env) 분리.
- **OCP** — 새 케이스 추가 시 fixture 디렉터리만 추가. 코드 수정 없음.
- **DIP** — Router 는 FixtureLoader 인터페이스에 의존. 테스트는 in-memory loader 주입.

## 3. 인터페이스

### 3.1 API 엔드포인트

```
GET /config/httpm?env={dev|stage|prod}[&case={case-id}]
```

| 파라미터 | 필수 | 기본값 | 설명 |
|---|---|---|---|
| `env` | ✅ | — | 환경 (`dev`, `stage`, `prod`) |
| `case` | ❌ | `case-001` | 픽스처 케이스 ID |

성공 응답 (200):
```json
{
  "env": "dev",
  "before": "<fixture content>",
  "after": "<fixture content>",
  "meta": {
    "fixture": "case-001",
    "version": "v1"
  }
}
```

에러 응답:
- `422`: `env` 파라미터 누락 (FastAPI 자동)
- `404`: case 또는 env fixture 미존재

```
GET /health
```
응답 (200): `{ "status": "ok" }`

### 3.2 데이터 모델 (`models.py`)

```python
from pydantic import BaseModel

class FixtureMeta(BaseModel):
    fixture: str        # case ID
    version: str        # 고정 "v1"

class ConfigResponse(BaseModel):
    env: str
    before: str
    after: str
    meta: FixtureMeta
```

> Q-01 결정: `meta.version` 은 코드에서 **"v1" 고정**. fixture 스키마 변경 시 버전업 — 현재 단일 버전.

### 3.3 FixtureLoader (`loader.py`)

```python
from pathlib import Path
from .models import ConfigResponse

class FixtureLoader:
    def __init__(self, fixtures_dir: Path) -> None:
        self._fixtures_dir = fixtures_dir

    def load(self, case: str, env: str) -> ConfigResponse:
        """fixture 파일에서 before/after 로드.

        - case 디렉터리 없음 → FixtureNotFoundError
        - env fixture 없음 → FixtureNotFoundError
        """

    def case_exists(self, case: str) -> bool: ...
    def env_exists(self, case: str, env: str) -> bool: ...
```

> Q-02 결정: fixture 인코딩 검증 **하지 않음** — UTF-8 가정 (요구사항 A-01). 파일 읽기 실패 시 500.

### 3.4 Settings (`settings.py`)

```python
from pydantic_settings import BaseSettings, SettingsConfigDict
from pydantic import Field

class LeebalsoSettings(BaseSettings):
    model_config = SettingsConfigDict(env_file=".env", extra="ignore")

    leebalso_fixtures_dir: str = Field(
        alias="LEEBALSO_FIXTURES_DIR", default="fixtures"

```

## 4. 핵심 시퀀스

```
client
  │
  │ GET /config/httpm?env=dev&case=case-001
  │
  ├─► app.router:
  │   - env 파라미터 검증 (FastAPI Query — 누락 시 422)
  │   - loader.load(case="case-001", env="dev")
  │
  ├─► FixtureLoader.load:
  │   - fixtures/case-001/ 존재? → no → 404
  │   - fixtures/case-001/dev.before.httpm 존재? → no → 404
  │   - 파일 읽기 → ConfigResponse(env, before, after, meta)
  │
  ▼
ConfigResponse(env="dev", before="...", after="...",
               meta={"fixture": "case-001", "version": "v1"})
```

## 9. 운영 / 배포 고려

- **배포 단위:** Docker 이미지 (`leebalso-mock:latest`).
- **컨테이너 이미지:** 표준 4-stage. runtime 은 uvicorn 으로 서버 기동.
- **헬스체크:** `GET /health` → `{"status": "ok"}`.
- **포트:** 9100 (architecture_test.md §3).

## 10. SOLID 검토

| 원칙 | 적용 |
|---|---|
| SRP | Router(라우팅) / FixtureLoader(파일 I/O) / Models(직렬화) / Settings(env) 분리 |
| OCP | 새 케이스 추가 = fixture 디렉터리만 추가. 코드 무수정 |
| LSP | FixtureLoader 를 다른 구현(in-memory, DB)으로 교체 가능 |
| ISP | 라우터는 loader.load() 만 호출 — 파일 I/O 세부 사항 불필요 |
| DIP | create_app(loader=...) 팩토리 주입. 테스트는 tmp_path fixture 주입 |
