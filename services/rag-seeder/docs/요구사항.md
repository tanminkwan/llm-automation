## 1. 개요

RAG 코퍼스를 Qdrant 벡터 DB 에 적재하는 **one-shot 컨테이너**. `corpus/**/*.comments.yaml` 파일을 읽어 각 snippet 의 주석(comment)을 `EmbeddingGateway` 로 벡터화한 뒤 Qdrant `code_comments` 컬렉션에 upsert 한다.

## 2. 배경 / 목적

워크플로우 A (코드 주석 자동화) 에서 RAG MCP 가 유사 코드/주석을 검색하려면, 사전에 코드 주석 쌍 데이터가 Qdrant 에 적재되어 있어야 한다. `rag-seeder` 는 스택 부팅 시 한 번 실행되어 코퍼스를 적재하고 종료하는 초기화 서비스다.

테스트 코퍼스 소스: https://github.com/tanminkwan/mwagent `src/main/java/mwagent/` Java 코드. 작업 완료 후 별도 repository 를 생성하여 seed-repo 로 사용한다.

## 3. 기능 요구사항 (FR)

| ID | 산출물 / 기능 | 설명 |
|---|---|---|
| FR-01 | YAML 코퍼스 로드 | `corpus/**/*.comments.yaml` 경로에서 snippet 목록을 재귀적으로 로드 |
| FR-02 | 임베딩 벡터화 | 각 snippet 의 `comment` 필드를 `EmbeddingGateway.embed()` 로 벡터 변환 (alias: `embedding`) |
| FR-03 | Qdrant upsert | `code_comments` 컬렉션에 벡터 + payload(id, path, symbol, line_range, comment, repo) 를 upsert |
| FR-04 | 컬렉션 자동 생성 | 대상 컬렉션이 없으면 `EMBEDDING_DIM` 에 맞춰 자동 생성 |
| FR-05 | 멱등 실행 | 같은 코퍼스를 재실행해도 중복 포인트가 생기지 않음 (deterministic ID 기반 upsert) |
| FR-06 | 배치 처리 | 대량 snippet 을 설정 가능한 배치 크기로 나누어 임베딩/upsert |
| FR-07 | 실행 결과 로깅 | 처리 건수(로드/임베딩/upsert), 소요 시간, 에러 건수를 stdout 로그로 출력 |

## 5. 제약 / 가정

- C-01: Qdrant 와 Embedding API 는 네트워크 도달 가능해야 한다 (런타임 의존).
- C-02: `corpus/` 디렉터리 구조는 `architecture_test.md §8.1` 형식을 따른다.
- A-01: 테스트 시 Qdrant/Embedding 은 Protocol 기반 fake/mock 으로 대체한다.
- A-02: 임베딩 차원은 `EMBEDDING_DIM=1024` (prod/test 동일).
