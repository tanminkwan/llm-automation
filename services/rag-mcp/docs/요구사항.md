## 1. 개요

RAG 벡터 검색을 MCP (Model Context Protocol) 도구로 노출하는 FastAPI 서비스. AI Agent(OpenCode 등)가 `search_codebase(query, k)` 를 호출하면, 쿼리를 임베딩하여 Qdrant `code_comments` 컬렉션에서 top-k 유사 코드 식별자와 주석을 반환한다.

## 2. 배경 / 목적

워크플로우 A 에서 AI Agent 가 코드 주석을 삽입할 때, 기존 코퍼스에서 유사한 코드/주석 쌍을 참고해야 한다. `rag-mcp` 는 이 검색 기능을 MCP tool call 형태로 제공하여, Agent 가 LLM 대화 루프 안에서 자연스럽게 호출할 수 있게 한다.

## 3. 기능 요구사항 (FR)

| ID | 산출물 / 기능 | 설명 |
|---|---|---|
| FR-01 | `search_codebase` MCP tool | `query`(str) + `k`(int, 기본값 설정 가능) 입력 → top-k 결과 반환 |
| FR-02 | 쿼리 임베딩 | `query` 를 `EmbeddingGateway.embed()` 로 벡터화 (alias: `embedding`) |
| FR-03 | Qdrant 벡터 검색 | `code_comments` 컬렉션에서 cosine similarity top-k 검색 |
| FR-04 | 결과 스키마 | 각 결과에 `id`, `path`, `symbol`, `line_range`, `comment`, `score` 포함 |
| FR-05 | 빈 결과 처리 | 컬렉션이 비어 있거나 검색 결과가 없으면 빈 리스트 반환 (에러 아님) |
| FR-06 | 헬스체크 | `GET /health` → `{"status": "ok"}` |
| FR-07 | k 범위 검증 | k ≤ 0 또는 상한 초과 시 적절한 에러 응답 |

## 5. 제약 / 가정

- C-01: Qdrant 와 Embedding API 는 네트워크 도달 가능해야 한다 (런타임 의존).
- C-02: `code_comments` 컬렉션은 `rag-seeder` 가 사전 생성/적재 완료한 상태를 가정.
- A-01: 테스트 시 Qdrant/Embedding 은 Protocol 기반 fake/mock 으로 대체한다.
- A-02: MCP 프로토콜은 FastAPI JSON-RPC 스타일 엔드포인트로 구현한다.

## 6. 영향 범위

- 상류: `libs/llm-gateway` (EmbeddingBackend Protocol), Qdrant (`code_comments` 컬렉션)
- 하류: AI Agent (OpenCode/Claude Code) — tool call 로 호출, `services/celery-comment` (Phase 5)
