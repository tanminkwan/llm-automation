## 3. API 엔드포인트

### POST /webhook/push

- HMAC 검증 (`X-Gitea-Signature`)
- 멱등키 확인 (`X-Gitea-Delivery`)
- work_type 판별 → Celery 발행
- 응답: `200 {"status": "accepted", "task_id": "..."}` 또는 `200 {"status": "skipped"}` (해당 없는 파일)

### GET /health

- 응답: `200 {"status": "ok"}`

## 4. 설정 항목

| 키 (env) | 의미 | 기본값 | 필수 | 민감 |
|---|---|---|---|---|
| `WEBHOOK_HOST` | 바인드 호스트 | `0.0.0.0` | N | N |
| `WEBHOOK_PORT` | 리스닝 포트 | `8080` | N | N |
| `WEBHOOK_SECRET` | HMAC 시크릿 | — | Y | Y |
| `CELERY_BROKER_URL` | Redis broker URL | `redis://localhost:6379/0` | N | N |
| `DEDUP_TTL_SECONDS` | 멱등키 캐시 TTL | `300` | N | N |

## 5. Celery Task 발행 규약

| work_type | task name | queue | args |
|---|---|---|---|
| `comment` | `comment.process` | `comment_queue` | `(work_id, repo_url, clone_url, ref, changed_files)` |
| `configaudit` | `configaudit.process` | `configaudit_queue` | `(work_id, repo_url, ref, changed_files)` |

- `work_id` = `{repo_full_name}:{after_sha[:8]}` (고유 식별자)
